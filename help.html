Overview

Add markers to a JSON file and show them on a map.

  • Markers are added using markers-editor.html which runs locally.
  • Edited marker file is temporarily saved to your Downloads folder and must be copied back to the project folder.
  • Use Visual Studio to push the updated project to GitHub.
  • Markers are shown on index.html at https://ladyerin42-boat.github.io/trip2025/index.html?nocache=1.

Adding markers to a map

Run the editor page locally

In a terminal, run the server.py script in the project folder to create a localhost server:

cd /path/to/project
python server.py

Or alternatively:

python -m http.server 8000

The page will be visible at: http://localhost:8000/marker-editor.html

Edit the current trip file

The page loads a default file (e.g., south_2025.json). Edit the defaultFile constant to work on a new trip.

Add entries by clicking on the map. Lat/Lng are automatically populated. Add the location name and the URL (Facebook post). Note: Facebook links must be public.

Save the markers by clicking the Download markers.json button. This:

  • Creates a brand-new file called [defaultFile].json
  • Saves it in your browser's Downloads folder
  • ❌ Does not overwrite the original JSON in your project or on GitHub

After downloading, you must:

  • Manually copy the new JSON file into your project folder (replacing the old one)
  • Commit and push it to GitHub so Pages can serve the updated markers

How to sync files to GitHub

In Visual Studio:

  • Open View → Git Changes
  • Changed files appear in the panel
  • Add a comment
  • Click Commit and Push

If successful, the changed files are synced to the GitHub project.

Viewing entries

The updated markers.json file is used by GitHub Pages to serve the page. After uploading, the new index.html page should be visible in a few seconds or minutes.

Page is served from: https://ladyerin42-boat.github.io/trip2025/

Try CTRL-SHIFT-R to force the browser to reload the page and bypass cache.

Or use this direct link with cache-busting: index.html?nocache=1
